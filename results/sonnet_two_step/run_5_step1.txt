`task_scheduler.py` written. Here's a summary of what's implemented:

**Core types**
- `TaskStatus` enum — `PENDING / RUNNING / COMPLETED / FAILED`
- `Task` dataclass — all required fields, validates `priority` in `__post_init__`
- `CircularDependencyError` / `TaskNotFoundError` — typed exceptions

**TaskScheduler**
| Feature | Implementation |
|---|---|
| Priority queue | Tasks sorted descending by `priority` within each dependency level |
| Dependency resolution | Kahn's algorithm (BFS topological sort) |
| Cycle detection | Raises `CircularDependencyError` with offending task IDs |
| Concurrent execution | `asyncio.gather` + `asyncio.Semaphore(max_concurrency)` |
| Retry / back-off | Exponential: `base_delay * 2^(attempt-1)`, up to `max_retries` |
| Observer events | `on_task_start`, `on_task_complete`, `on_task_fail` via async `EventEmitter` |
| Execution plan | `get_execution_plan()` → `List[List[Task]]` (no side effects) |
| Metrics | `SchedulerMetrics` / `TaskMetrics` — per-task start/end/elapsed + retry count + total scheduler wall time |

**Design notes**
- A task whose dependency has failed is immediately marked `FAILED` and skipped, but sibling branches continue unaffected.
- All event handlers are `async`; `emit` awaits each one in registration order.
- `run()` returns the fully populated `SchedulerMetrics` object.

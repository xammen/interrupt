====================================================================================================
     THE INTERRUPT EFFECT - Experimental Results (5 Conditions)
     Does splitting code generation from test generation change what LLMs produce?
====================================================================================================

METHODOLOGY:
  - Same core task for all runs (async task scheduler with tests)
  - 5 conditions per model, 5 runs each = 50 total runs
  - COMPLETE:     Full single-prompt generation (code + tests)
  - INTERRUPTED:  Generation killed at 20s, then fresh re-generation
  - FRAGMENT:     Complete run cut at 50% of lines, fragment fed back to continue
  - TWO_STEP:     Step 1: code only. Step 2: given code, write tests
  - CODE_GIVEN:   Given complete code from condition A, write tests
  - Models: Claude Sonnet 4.6, Claude Opus 4.6 (via Anthropic provider)

====================================================================================================
  CONCLUSIONS
====================================================================================================

  1. INTERRUPTION ALONE DOES NOTHING
     Complete ~= Interrupted (p > 0.05 for both models)

  2. TASK SEPARATION PRODUCES 2x-5x MORE TESTS
     Fragment, Two-Step, and Code-Given all produce massively more tests
     than single-prompt Complete (p < 0.01, r = 1.00, zero overlap)

  3. THE MECHANISM IS TASK ISOLATION, NOT SELF-REVIEW
     Code-Given (model sees someone else's code) = Two-Step (model sees its own)
     The model doesn't benefit from reviewing its own output specifically

  4. OPUS IS MORE AFFECTED THAN SONNET
     Opus single-prompt: ~10 tests. Opus two-step: ~53 tests (5x increase)
     Sonnet single-prompt: ~22 tests. Sonnet two-step: ~51 tests (2.3x increase)

  5. PROMPT ARCHITECTURE > MODEL CAPABILITY
     Behavior is a property of context, not of the model. The same model
     produces 10 or 53 tests depending purely on how you structure the prompt.
